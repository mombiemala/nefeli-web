# ✶ Luminary

**A deeply personal AI astrology companion.**

Luminary is not a generic horoscope app. It connects your birth chart to your
_lived experience_ — your healing journey, career, creative practice, and
relationships — and interprets every planetary event through **that** lens.
It remembers what you told it: the toxic pattern you declared freedom from last
month, the job search while caregiving, the unfinished paintings — and reads the
sky through your actual story, powered by persistent memory + Claude.

---

## Highlights

- **Onboarding** — birth data collection (with "I don't know my birth time"
  handling), life-context capture across 7 categories, a healing focus, a
  declaration, then an animated chart reveal + a personalised 3-paragraph
  welcome reading that integrates your chart with what you just shared.
- **Birth Chart Explorer** — an interactive SVG chart wheel (pure React/SVG,
  no chart lib), planet table, aspect grid, big-three + elemental balance, and a
  per-planet slide-over with a Claude reading of _how that placement lives in
  your specific life_.
- **Today's Guidance** — moon sign/phase, an energy badge (Power Day / Navigate
  / Move Gently / Rest), the day's key transits, a personal paragraph, a
  journaling prompt, and the next 7 days — cached per day.
- **Luminary Chat** — streaming conversational AI with your full chart, live
  transits, moon phase, life context, declarations and recent insights injected
  into a dynamically-assembled system prompt. Conversation starters, persistent
  history, and a "Remember this" button that saves insights.
- **Monthly Guide** — month overview, moon phases, a major-transits accordion
  with a "For you specifically" section per event, and a day-by-day table.
- **Transit Tracker** — all current transits, filterable (active / upcoming /
  high-intensity) and sortable (intensity / date / house), each with an
  intensity rating and an "Explore" link into chat.
- **Life Context Manager** — add / archive context entries and declarations,
  plus an insights timeline.

## Getting started

```bash
# 1. Install
npm install

# 2. Configure
cp .env.example .env      # fill in DATABASE_URL + secrets (or leave demo mode on)

# 3. Database
npx prisma db push

# 4. Run
npm run dev               # http://localhost:3000
```

### Demo mode (zero API keys)

With `LUMINARY_DEMO_MODE="true"` (the default in `.env.example`), Luminary runs
end-to-end with **no external keys**:

- Charts, transits and moon phases are generated by a deterministic local
  pseudo-ephemeris (`lib/demo-data.ts`) — stable per birth profile.
- Claude replies are stand-ins that reflect your assembled context back to you.
- Geocoding uses a built-in city table + a deterministic fallback.
- A **"Continue in demo mode"** button on the login screen signs you straight in.

Add `ANTHROPIC_API_KEY`, `RAPIDAPI_KEY` and `OPENCAGE_API_KEY` to switch to live
data — no code changes needed. Demo mode also auto-enables if core keys are
missing.

## Environment variables

See [`.env.example`](./.env.example). Core: `DATABASE_URL`, `NEXTAUTH_SECRET`,
`NEXTAUTH_URL`. Live data: `RAPIDAPI_KEY`, `OPENCAGE_API_KEY`,
`ANTHROPIC_API_KEY`. Email magic links: `EMAIL_SERVER_*` / `EMAIL_FROM`
(when unset, the link is printed to the server console). Cron: `CRON_SECRET`.

## The Claude system prompt

Every call assembles a system prompt (`lib/claude.ts` + `lib/luminary-context.ts`)
containing: the natal-chart AI-context XML, current-transit XML, moon phase, a
concise life-context summary, active declarations, the last 5 insights, and the
current date + location — so guidance is always specific and time-aware.

## Deployment

Deploy to Vercel with a PostgreSQL database (e.g. Vercel Postgres / Neon /
Supabase). `vercel.json` schedules the daily-guidance cron at 05:00 UTC.
